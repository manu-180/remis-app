# Naming — App de Remisería

## Contexto de producto

Aplicación single-tenant para una remisería en la zona de Santa Rosa, La Pampa (código área 2954). Opera con asignación manual por despachante, tarifa regulada por ordenanza municipal y una flota de aproximadamente 50 conductores. La identidad visual apunta a "Premium Pampeano": sobria, confiable, con regionalidad sutil, sin folclore explícito.

---

## Criterios de evaluación

Cada candidato fue analizado contra estos filtros:

- **Pronunciación**: natural en español rioplatense, sin ambigüedades
- **CTA verbal**: "Pedí un [NOMBRE]" tiene que sonar fluido
- **Longitud**: menos de 10 letras
- **Identidad visual**: si el nombre sugiere un símbolo o ícono natural
- **Riesgo de colisión**: similitud con apps de remis, taxi o movilidad existentes
- **Disponibilidad de dominio**: candidatos para chequeo posterior en `.com.ar` y `.com`
- **App Store**: sin colisión evidente con apps de transporte ya publicadas

No se incluyen referencias a "Uber", "Cab" ni "Taxi" —ni directas ni irónicas.

---

## Familias exploradas

Las familias de nombres cubiertas son:

1. **Verbos cortos** (acción directa)
2. **Sustantivos pampa-tinged** (sin folclore)
3. **Compuestos sintéticos** (palabras construidas)
4. **Conceptos de servicio** (abstractos funcionales)
5. **Localización con sufijo** (nombre de pueblo + extensión)

## Advertencias específicas

- **Vamos**: verificación urgente antes de avanzar — alta probabilidad de que el nombre esté tomado en App Store Argentina.
- **Cardo**: la connotación negativa en castellano ("persona difícil") es un riesgo real para una app de atención al pasajero.
- **Traza**: el género gramatical hace que el CTA "Pedí un Traza" sea incómodo.
- **Nombres con tilde en última sílaba** (Caldén, Chofé, Vialá): los dominios `.com.ar` se pueden registrar con o sin tilde, pero los usuarios van a buscar sin tilde — conviene registrar ambas versiones.

## Nomenclatura técnica del proyecto

Independiente del nombre comercial que se elija, el proyecto técnico mantiene esta nomenclatura:

| Artefacto | Valor |
|-----------|-------|
| Código de repositorio | `remis-app` |
| Bundle ID — app conductor | `com.[cliente].remis.driver` |
| Bundle ID — app pasajero | `com.[cliente].remis.passenger` |
| Display name en stores | Definir post-naming |
| Slug técnico | Definir post-naming (ej. `calden`, `sereno`) |

El placeholder `[cliente]` se reemplaza con el identificador que elija el cliente (generalmente nombre de empresa o razón social abreviada).

## Proceso sugerido de decisión

Una vez revisado este documento, se recomienda el siguiente proceso para elegir el nombre final:

### Paso 1 — Filtrado interno (1 semana)
El cliente y el equipo operativo descartan candidatos por preferencia subjetiva o restricciones que solo ellos conocen (nombres de competidores locales, connotaciones en el pueblo, etc.). Objetivo: quedar con 3 o 4 finalistas.

### Paso 2 — Verificación de disponibilidad
Para cada finalista, chequear:
- `.com.ar` en [nic.ar](https://nic.ar)
- `.com` en cualquier registrador (Namecheap, GoDaddy, Porkbun)
- App Store: buscar el nombre en App Store Connect > My Apps > "Availability Check" o simplemente buscarlo en la App Store pública
- Google Play: buscar en la Play Store para detectar colisiones obvias

### Paso 3 — Test con despachantes y conductores
Hacer una prueba informal con 5 a 10 empleados: mostrar los 3 finalistas y preguntar cuál les resulta más natural para decirle a un pasajero. No hace falta encuesta formal — conversación directa alcanza.

### Paso 4 — Decisión y registro
Una vez elegido el nombre:
1. Registrar dominio `.com.ar` (y `.com` si está disponible)
2. Confirmar disponibilidad de nombre en App Store Connect antes de diseñar el ícono
3. Informar al equipo técnico para configurar Display Names y Bundle IDs definitivos

### Paso 5 — Brief de identidad visual
Con el nombre elegido, pasar a brief de marca: paleta, tipografía, ícono. El nombre define si el símbolo es figurativo (árbol, planta) o abstracto (trayecto, movimiento).
